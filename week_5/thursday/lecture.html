What are HTML Forms?

Forms are interactive sections of web pages that collect user input.

Common Uses:

  • Login and registration
  • Contact forms
  • Surveys and feedback
  • Shopping checkouts
  • Search boxes
  • Comments and reviews

Basic Form Structure:

<form action="/submit" method="post"> <!-- Form controls go here --> </form>

Key Point: Forms are containers for various input controls

Important Form Attributes

Attribute Purpose Example
required Field must be filled <input required>
disabled Field cannot be edited <input disabled>
readonly Can see but not edit <input readonly>
placeholder Hint text <input placeholder="Enter name">
value Default value <input value="Default">
maxlength Character limit <input maxlength="10">
pattern Regex validation <input pattern="[A-Z]{3}">

These attributes provide built-in HTML5 validation!

Accessing Forms with JavaScript

Getting Form Elements

// Get the form let form = document.getElementById('myForm'); // Get input values let username = document.getElementById('username').value; let email = document.getElementById('email').value; // Get selected option let selectBox = document.getElementById('country'); let selectedValue = selectBox.value; let selectedText = selectBox.options[selectBox.selectedIndex].text; // Get checked checkboxes let checkbox = document.getElementById('agree'); if (checkbox.checked) { console.log('User agreed!'); }

Remember: .value gets the current value of any input field

11 / 18

Form Events

Common Form Events:

  • submit - Form is submitted
  • change - Input value changes
  • input - User types in field
  • focus - Field gets focus
  • blur - Field loses focus

Handling Form Submission

let form = document.getElementById('myForm'); form.addEventListener('submit', function(event) { // Prevent default submission event.preventDefault(); // Get form data let name = document.getElementById('name').value; // Process the data console.log('Form submitted with name:', name); });

Important: preventDefault() stops the form from refreshing the page!

Real-time Validation

Validating as User Types

let emailInput = document.getElementById('email'); let errorMsg = document.getElementById('emailError'); emailInput.addEventListener('input', function() { let email = emailInput.value; // Simple email validation if (!email.includes('@')) { errorMsg.textContent = 'Please include @ in email'; errorMsg.style.color = 'red'; } else { errorMsg.textContent = '✓ Valid email'; errorMsg.style.color = 'green'; } });

Password Strength Checker

function checkPasswordStrength(password) { if (password.length < 6) return 'Weak'; if (password.length < 10) return 'Medium'; return 'Strong'; }

Complete Validation Example

function validateForm() { let isValid = true; // Check name let name = document.getElementById('name').value; if (name.length < 2) { showError('name', 'Name must be at least 2 characters'); isValid = false; } // Check email let email = document.getElementById('email').value; if (!email.includes('@') || !email.includes('.')) { showError('email', 'Please enter a valid email'); isValid = false; } // Check if terms accepted let terms = document.getElementById('terms'); if (!terms.checked) { showError('terms', 'You must accept the terms'); isValid = false; } return isValid; }

Modern Approach: FormData API

Easier Way to Get All Form Data

let form = document.getElementById('myForm'); form.addEventListener('submit', function(event) { event.preventDefault(); // Get all form data at once let formData = new FormData(form); // Convert to object let data = {}; formData.forEach((value, key) => { data[key] = value; }); console.log('All form data:', data); });

FormData automatically collects all named form fields!

Form Best Practices

✅ DO:

  • Always use labels with correct 'for' attributes
  • Group related fields with fieldset
  • Provide immediate feedback for validation
  • Use preventDefault() when handling forms with JS
  • Add placeholder text for guidance
  • Make forms keyboard accessible
  • Use appropriate input types (email, tel, date)

❌ DON'T:

  • Rely only on client-side validation
  • Make all fields required unnecessarily
  • Use placeholders instead of labels
  • Forget to test form on mobile devices

Golden Rule: Always validate on both client AND server!
