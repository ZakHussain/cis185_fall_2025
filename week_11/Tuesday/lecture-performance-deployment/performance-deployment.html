Image Optimization

The Biggest Performance Win

Images often account for 50-90% of page weight!

<!-- Use modern formats --> <picture> <source srcset="image.webp" type="image/webp"> <source srcset="image.jpg" type="image/jpeg"> <img src="image.jpg" alt="Description"> </picture> <!-- Lazy loading --> <img src="photo.jpg" loading="lazy" alt="Photo"> <!-- Responsive images --> <img srcset="small.jpg 300w, medium.jpg 600w, large.jpg 1200w" sizes="(max-width: 600px) 300px, 600px" src="medium.jpg" alt="Responsive">

CSS Optimization

Faster Styling

/* 1. Critical CSS - Inline above-the-fold styles */ <style> /* Only styles needed for initial view */ body { margin: 0; font-family: sans-serif; } .header { background: #333; color: white; } </style> /* 2. Defer non-critical CSS */ <link rel="preload" href="styles.css" as="style" onload="this.rel='stylesheet'"> /* 3. Minify CSS */ Remove whitespace, comments, shorten names style.css: 50KB ‚Üí style.min.css: 35KB /* 4. Remove unused CSS */ Tools: PurgeCSS, UnCSS

Caching Strategies

Don't Re-download What You Have

# Browser Caching (set in server config) # Cache-Control header examples: # Static assets (cache for 1 year) Cache-Control: public, max-age=31536000, immutable # HTML files (always revalidate) Cache-Control: no-cache # Service Workers (offline caching) self.addEventListener('fetch', event => { event.respondWith( caches.match(event.request) .then(response => response || fetch(event.request)) ); });

Cache Busting: Use versioned filenames (app.v2.js) or hashes (app.abc123.js)

Environment Variables

Managing Configuration Securely

# .env file (local development) VITE_API_URL=https://api.example.com VITE_API_KEY=your-secret-key # Access in React/Vite const apiUrl = import.meta.env.VITE_API_URL; # IMPORTANT: Add to .gitignore .env .env.local .env.*.local # Set in deployment platform: Netlify: Site settings ‚Üí Environment variables Vercel: Project settings ‚Üí Environment Variables

Never commit secrets! API keys, passwords, and tokens should always be in environment variables, never in code.

Portfolio Best Practices

Making It Professional

  • Mobile Responsive: Test on all devices
  • Fast Loading: Under 3 seconds
  • Clean Design: Professional, not flashy
  • Working Links: Test all project links
  • Spell Check: No typos or grammar errors
  • Updated: Remove outdated projects
  • Accessible: Good contrast, alt text
  • Personal Touch: Show your personality

Performance Check: Run Lighthouse on your portfolio. Aim for 90+ scores!
