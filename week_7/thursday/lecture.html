Setting SMART Goals for Your Project

SMART What it Means Your Project Example
Specific Clear and detailed Build a 4-page coffee shop website
Measurable Can track progress Complete 1 page per week
Achievable Realistic for you Use HTML/CSS skills from class

Good Goal: "Create a 4-page responsive portfolio by March 15th with contact form"

Step 5: Testing Your Website

Make Sure Everything Works!

Testing Checklist:

  • All links work (no broken links)
  • Images load properly
  • Forms submit correctly
  • Works on phone, tablet, and desktop
  • Works in Chrome, Firefox, Safari
  • No spelling or grammar errors
  • JavaScript features work without errors
  • Pages load quickly (under 3 seconds)

How to Test:

  • üì± Mobile Test: Use browser's device mode (F12)
  • üë• User Test: Ask a friend to try your site
  • üîç Link Test: Click every link
  • üìù Form Test: Submit test data
  • üåê Browser Test: Open in different browsers

Pro Tip: Test as you build, not just at the end!

Common Beginner Mistakes (And How to Avoid Them)

Learn from Others' Mistakes!

‚ùå Mistake ‚úÖ Better Approach
Starting with complex features Build basic structure first
Not planning before coding Sketch wireframes first
Messy file organization Use clear folder structure
Forgetting mobile users Test on phone regularly
Huge image files Optimize images (under 200KB)
No backup copies Save versions as you go

Time Management Tips:

  • Don't try to learn everything at once
  • Use code you understand (don't just copy-paste)
  • Ask for help when stuck for more than 30 minutes
  • Save working versions before adding new features
