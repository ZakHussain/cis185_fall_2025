What is React?

A JavaScript Library for Building User Interfaces

React is an open-source JavaScript library created by Facebook (Meta) in 2013.

  • Component-based: Build encapsulated pieces that manage their own state
  • Declarative: Describe what UI should look like, React handles the DOM
  • Learn once, write anywhere: React Native for mobile, React for web

Fun Fact: React powers Facebook, Instagram, Netflix, Airbnb, and many more major applications.

The Virtual DOM

React's Secret Sauce

Traditional DOM manipulation is slow. React uses a Virtual DOM to optimize updates:

  1. React keeps a lightweight copy of the DOM in memory
  2. When state changes, React creates a new Virtual DOM
  3. React compares old vs new (called "diffing")
  4. Only the differences are applied to the real DOM

Result: Faster updates and better performance!

Components

The Building Blocks of React

Components are independent, reusable pieces of UI. Think of them like LEGO blocks:

// A simple component function Welcome() { return <h1>Hello, React!</h1>; } // Using the component <Welcome />

Key Concept: Components can be nested, reused, and composed together to build complex UIs.

Types of Components

Function Components vs Class Components

// Function Component (Modern - Preferred) function Greeting() { return <p>Hello from function component!</p>; } // Class Component (Legacy) class Greeting extends React.Component { render() { return <p>Hello from class component!</p>; } }

Modern React: Function components with Hooks are the standard. We'll focus on these.

JSX - JavaScript XML

Writing HTML in JavaScript

JSX is a syntax extension that looks like HTML but compiles to JavaScript:

// This JSX: const element = <h1>Hello, world!</h1>; // Compiles to this JavaScript: const element = React.createElement( 'h1', null, 'Hello, world!' );

Why JSX? It makes React code more readable and easier to write.

JSX Rules

Important Syntax Differences

// 1. Must return a single parent element return ( <div> <h1>Title</h1> <p>Content</p> </div> ); // 2. Use className instead of class <div className="container"> // 3. Use camelCase for attributes <button onClick={handleClick}> // 4. Self-closing tags must end with /> <img src="pic.jpg" />

Rendering Lists

Using map() to Display Arrays

function ShoppingList() { const items = ["Apples", "Bananas", "Oranges"]; return ( <ul> {items.map((item, index) => ( <li key={index}>{item}</li> ))} </ul> ); }

Important: Always provide a unique key prop when rendering lists. It helps React identify which items changed.

Props (Properties)

Passing Data to Components

Props allow you to pass data from parent to child components:

// Parent component passes props function App() { return ( <div> <Greeting name="Alice" age={25} /> <Greeting name="Bob" age={30} /> </div> ); } // Child component receives props function Greeting(props) { return <h1>Hello, {props.name}! Age: {props.age}</h1>; }
14 / 24

Destructuring Props

Cleaner Syntax for Props

// Instead of using props.name, props.age... function Greeting(props) { return <h1>Hello, {props.name}!</h1>; } // Use destructuring for cleaner code function Greeting({ name, age }) { return ( <div> <h1>Hello, {name}!</h1> <p>You are {age} years old.</p> </div> ); }

Best Practice: Use destructuring to make your components more readable.
