HTTP Requests and Responses

Client-Server Communication

The Fetch API

Making HTTP Requests in JavaScript

The Fetch API provides a modern way to make HTTP requests:

// Basic fetch request fetch("https://swapi.dev/api/people/1/") .then(response => response.json()) .then(data => { console.log(data); // Process the data }) .catch(error => { console.error("Error:", error); });

Key Points:

  • fetch() returns a Promise
  • .then() handles the response
  • .json() parses JSON data
  • .catch() handles errors

Promises and Async Operations

Handling Asynchronous Code

JavaScript is asynchronous - it doesn't wait for long operations to complete.

// Promises have three states: // 1. Pending - waiting for result // 2. Fulfilled - operation succeeded // 3. Rejected - operation failed let promise = fetch("https://api.example.com/data"); promise .then(response => { // This runs when fulfilled return response.json(); }) .catch(error => { // This runs when rejected console.error(error); });

Why Async? Network requests take time. Promises let your code continue running while waiting for the response!

Async/Await

Cleaner Asynchronous Code

The async/await syntax makes asynchronous code look synchronous:

// Using .then() chains fetch("https://swapi.dev/api/people/1/") .then(response => response.json()) .then(data => console.log(data)) .catch(error => console.error(error)); // Using async/await (cleaner!) async function getData() { try { let response = await fetch("https://swapi.dev/api/people/1/"); let data = await response.json(); console.log(data); } catch (error) { console.error(error); } } getData();

Error Handling

What Happens When Things Go Wrong?

async function fetchWithErrorHandling(url) { try { let response = await fetch(url); // Check if response is OK (status 200-299) if (!response.ok) { throw new Error(`HTTP error! Status: ${response.status}`); } let data = await response.json(); return data; } catch (error) { // Handle network errors, invalid JSON, etc. console.error("Fetch error:", error.message); return null; } }

Common Errors:

  • Network error (no internet connection)
  • 404 - Resource not found
