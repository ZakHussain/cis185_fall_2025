Evolution of the Web

From Static to Dynamic

Early Web Pages (1990s)

  • Mostly text with hyperlinks - Think early Wikipedia, academic papers, documentation sites
  • Static content - same for everyone - Every visitor saw identical content, like a digital brochure
  • Simple "Hypertext" documents - Basic linking between pages, similar to clicking through a PowerPoint
  • Limited images and no interactivity - Small GIFs, no videos, no user input beyond clicking links

Modern Web Applications

  • Rich multimedia content - YouTube videos, Spotify streaming, Instagram galleries, interactive maps
  • Dynamic and personalized experiences - Amazon recommendations, Netflix homepage, Facebook feed tailored to you
  • Interactive user interfaces - Gmail drag-and-drop, Google Docs real-time editing, online banking transactions
  • Server-side processing and databases - Shopping carts remembering items, user accounts, search results from billions of pages

Client-Server Architecture

How Web Pages Are Delivered

The Request-Response Cycle:

  1. Client (Browser) sends HTTP request
  2. Web Server processes the request
  3. Server may connect to Application/Database servers
  4. Server sends HTTP response with HTML
  5. Browser renders the HTML page

Key Components:

  • Web Server: Stores and delivers web content
  • Application Server: Runs server-side code
  • Database Server: Manages data storage

Block Elements

  • Start on a new line
  • Take full width available
  • Stack vertically

Try This Code:

Inline Elements

  • Flow with text
  • Only take needed width
  • Stay on same line

Try This Code:

and this is <em>italic</em>.</p>

<p>Visit <a href="#">this link</a> or

<span>Span 2</span>

Styling Hook:

The primary purpose of <span> is to serve as a hook for applying CSS styles or manipulating content with JavaScript. By assigning class or id attributes to a <span>, specific styling rules can be targeted to that portion of the text or content.

<p>This is some <span style="color: red;">important</span> text.</p>

Nesting Rules: Block vs Inline

Understanding What Can Go Inside What

Rule 1: Inline Elements Inside Block Elements ‚úì

Valid:

<p>This is a paragraph with a <strong>bold word</strong> inside.</p>

This works because inline elements naturally fit within block-level containers.

Rule 2: Block Elements Cannot Go Inside Inline ‚úó

Invalid:

<strong><p>This is a bold tag with a paragraph inside it.</p></strong>

This is incorrect because a block-level <p> element cannot be placed inside an inline <strong> element.

Rule 3: Block Elements Can Contain Both

Block-level elements can contain both inline and other block-level elements. They typically start on a new line and take up the full width of their container.

Valid - Block inside Block:

<div><p>This is a paragraph inside a div.</p></div>

Valid - Mixed nesting:

<div><p>This paragraph contains a <span>nested span</span>.</p></div>

More Examples:

‚úì Correct Nesting

  <p>Text with <em>emphasis</em></p>

  </ul>
</div>

Semantic HTML5 Elements

What is Semantic HTML?

The purpose of semantic HTML is to add meaning and structure to web page content, improving accessibility for assistive technologies, enhancing search engine optimization (SEO) for better indexing, and increasing code maintainability by making it more readable and understandable for developers. By using tags like <nav>, <article>, and <section>, you communicate the content's role and intent, which helps browsers, machines, and users alike.

Common Semantic Elements:

  • <nav> - Navigation links
  • <header> - Page or section header
  • <main> - Main content area
  • <article> - Self-contained content
  • <section> - Thematic grouping
  • <aside> - Sidebar content
  • <footer> - Page or section footer

Working with Images

The Image Element

<img src="path/to/image.jpg" alt="Description">

Required Attributes:

  • src - Path to image file
  • alt - Alternative text (accessibility)

Image Sources - Local vs URL:

Local Image (in your project folder):

<img src="images/logo.png" alt="Company Logo">
<img src="../photos/profile.jpg" alt="Profile Photo">

External Image (from the web):

<img src="https://example.com/image.jpg" alt="Example Image">
<img src="https://picsum.photos/200/300" alt="Random Placeholder">

Optional Attributes:

  • width and height - Dimensions (in pixels or %)
  • title - Tooltip text on hover
  • loading="lazy" - Defer loading until needed
Notes:
  • <img> is an inline element and self-closing!
  • Using external URLs means relying on another server - images may disappear
  • Local images are more reliable but increase your project size

CSS Fundamentals

Styling Your HTML

CSS Rule Structure

selector {
    property: value;
    property: value;
}

Types of Selectors

Selector Type Example Targets
Element p { } All <p> elements
Class .highlight { } class="highlight"
ID #header { } id="header"

Example: Using <style> Tag in HTML

<!DOCTYPE html>
<html>
<head>
    <style>
        /* Target ID attribute */
        #header {
            background-color: blue;
            color: white;
            padding: 20px;
        }
        
        /* Target class attribute */
        .highlight {
            background-color: yellow;
            font-weight: bold;
        }
        
        /* Target element */
        p {
            font-size: 16px;
            line-height: 1.5;
        }
    </style>
</head>
<body>
    <div id="header">Welcome to My Site</div>
    <p class="highlight">This text is highlighted</p>
    <p>This is a regular paragraph</p>
</body>
</html>

The <head> Section

Page Metadata

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="description" content="Page description">
    
    <title>Page Title (appears in tab)</title>
    
    <link rel="stylesheet" href="styles.css">
    <link rel="icon" href="favicon.ico">
    
    <script src="script.js" defer></script>
</head>

Common Head Elements:

  • <title> - Browser tab title
  • <meta> - Page metadata
  • <link> - External resources
  • <style> - Internal CSS
  • <script> - JavaScript

Why Metadata is Important:

  • Search Engine Optimization (SEO): Meta descriptions appear in search results and help search engines understand your page content
  • Social Media Sharing: Open Graph meta tags control how your page appears when shared on Facebook, Twitter, etc.
  • Mobile Responsiveness: The viewport meta tag ensures proper scaling on mobile devices
  • Character Encoding: UTF-8 charset prevents text display issues with special characters and emojis
  • Browser Compatibility: Proper metadata helps browsers render your page correctly across different devices and screen sizes
  • Accessibility: Metadata helps screen readers and assistive technologies understand page structure and content

HTML Best Practices

Code Quality

  • Always use lowercase for tags and attributes
  • Quote all attribute values
  • Close all tags (even self-closing in XHTML)
  • Indent nested elements consistently
  • Use semantic HTML elements

Accessibility

  • Always include alt text for images
  • Use proper heading hierarchy (h1‚Üíh2‚Üíh3)
  • Ensure color contrast meets WCAG standards
  • Make forms accessible with labels

Performance

  • Optimize images for web
  • Minify CSS and JavaScript
  • Use appropriate image formats (WebP, JPEG, PNG)

Creating a Simple HTML Page

Step-by-Step Guide

Follow These Steps:

  1. Open a text editor - Use Notepad, VS Code, Sublime Text, or any plain text editor
  2. Start with the DOCTYPE - Type <!DOCTYPE html> at the very top
  3. Add the html element - Include <html lang="en"> to wrap everything
  4. Create the head section - Add <head> with charset and title
  5. Create the body section - Add <body> for visible content
  6. Add your content - Insert headings, paragraphs, and other elements
  7. Close all tags - Make sure every opening tag has a closing tag
  8. Save as .html - Save the file with a .html extension (e.g., index.html)
  9. Open in browser - Double-click the file or drag it into a web browser

Complete Example:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My First Web Page</title>
</head>
<body>
    <h1>Welcome to My Website!</h1>
    <p>This is my first HTML page.</p>
    <p>I'm learning web development!</p>
</body>
</html>
Pro Tip: Save your file in a dedicated folder for your web project. This makes it easier to organize images, CSS files, and other resources later.

Adding CSS with the <style> Tag

Step-by-Step Guide

Follow These Steps:

  1. Locate the <head> section - Find the opening and closing head tags
  2. Add the <style> tag - Place it between the <head> tags, after the title
  3. Write your CSS rules - Use selectors to target HTML elements
  4. Close the <style> tag - Don't forget the closing </style> tag
  5. Add classes/IDs to HTML - Add class or id attributes to elements you want to style
  6. Save and refresh - Save your file and refresh the browser to see changes

Before (No Styling):

<head>
    <meta charset="UTF-8">
    <title>My First Web Page</title>
</head>

After (With Styling):

<head>
    <meta charset="UTF-8">
    <title>My First Web Page</title>
    <style>
        body {
            font-family: Arial, sans-serif;
            margin: 40px;
            background-color: #f5f5f5;
        }
        
        h1 {
            color: #2a2f3c;
            text-align: center;
        }
        
        .highlight {
            background-color: yellow;
            padding: 10px;
        }
        
        #special-paragraph {
            color: blue;
            font-size: 18px;
        }
    </style>
</head>
<body>
    <h1>Welcome to My Website!</h1>
    <p class="highlight">This paragraph is highlighted.</p>
    <p id="special-paragraph">This paragraph is special.</p>
</body>
